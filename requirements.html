Centralized Food Ordering Portal: Virginia Tech

Requirements


Functional Requirements

FR-1. The system must support and authenticate the login for dining hall managers, southgate manager and vendors.

FR-2. The system must support the ability to check the availability of the products in the portal.

FR-3. The system must support the ability to place orders for products and add or delete the number of items required.

FR-4. The system must support the ability to request for items that are unavailable but necessary.

FR-5. The system must support the ability to retrieve the orders and place them in the portal.

FR-6. The system must send confirmation of items once the orders have been accepted by the vendors.

FR-7. The system must support the ability to update the portal to reflect the availability of the items.

FR-8. The system must support the ability to give confirmation of delivery to the dining hall managers on the portal.


Non-Functional Requirements

NFR-1. The login credentials for dining hall managers, southgate managers and vendors shall be added to the system.

NFR-2. Background check of the vendors must be done before adding them as vendors to the system.

NFR-3. The system should be scalable in order to support more and more vendors.

NFR-4. The system should be capable of supporting multiple dining hall managers logged in at the same time.

NFR-5. Each order request should be processed within 2 seconds.

NFR-6. The site should load within 3 seconds when the number of simultaneous users is > 10.

NFR-7. The website should be supported on different browsers.

NFR-8. The date format must be as follows: month/date/year.

NFR-9. There should be a dedicated printer so that the southgate manager can print the labels for the corresponding orders.
